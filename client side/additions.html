Cached data system

At UpComingEvents.com, we used the ticketmaster.com API to get information about the artist, including their photo and upcoming events. To obtain information about an artist or upcoming events, you need the artist ID, which can be obtained by sending an API request to them with the artist's name. This ID is then used for the rest of their API. To reduce the number of API requests (as there is a limit), we cached the artist ID and artist information. This way, when a get request is made, the data reaches the user faster without the need to send a new request to UpComingEvents. Additionally, every day, the cached data that has not been used in the last 7 days is removed to save space in the RAM.

Artists

At artist-details.html, we add the available/known social media accounts of the artist, as well as platforms where users can listen to this artist's music (only the available platforms will appear). We also provide additional information for the user, such as whether this artist is currently on tour and the upcoming events for this artist (date, location, status, ticket price). Additionally, we include a button that redirects the user to ticketmaster.com, where they can directly purchase tickets for these events.

Song

At song-details.html, we add the song and artist page on LastFM. We also include the name of the album that this song is a part of, along with a link to the album page on LastFM. Additionally, we provide some information about the song and include the YouTube video of the song.

Account

At account creation, we gave the user the ability to set his profile photo and add additional information. He can change it anytime in my account page, including email and password.
